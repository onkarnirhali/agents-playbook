# Coding Standards

## General

1. Keep changes minimal and goal aligned.
2. Preserve backwards compatibility unless the goal explicitly permits breaking changes.
3. Prefer explicit error handling and observable failures.
4. Keep comments concise and meaningful.

## Review Expectations

1. Include tests or clear rationale for missing tests.
2. Update goal tracking documents for meaningful changes.
3. Avoid hidden side effects.

## Documentation

1. Update related docs when behavior changes.
2. Keep internal links valid.
